Background: The university of Texas operates with a homegrown system and a highly customized stack of in-house tools and frameworks.

The project: Adding a modernized, Python/Django front end to the legacy mainframe course inventory system.

The legacy system: The back end is a mainframe coded with Natural/ADABAS, a procedural language similar to COBOL. Up until 2016, the only way for registrar employees to view the course inventory was using a maiframe interface. Coding the programs that ran each of the mainframe interfaces was a skill of its own. An example of the legacy system is pictured.

The new system: A familiar interface designed with Python and Django, greatly expanding the functionality available to users, including a variety of search parameters for the course listing, drilldown behavior to view to course details from the listing, a comparison function to view and compare semester data for a single course, and a change log. The back end remained on the mainframe as the registrar's data needed to remain accessible to all other departments while systems were being moved off of the mainframe campus-wide. The change log required a new ADABAS file, which underwent a design and approval process before its creation.

Background:The legacy system keeping the university grades process running smoothly is a combination of batch processing, web interfaces, and mainframe programs. Some of the programs date back to the mid 80's (one was older than I am), making the system among the more brittle at the university, as upgrades or changes must be made with care not to disrupt functionality.

It is the job of the grades steward to stay on the last day of the semester, when all grades have been turned in, to run the series of batch programs that store students' grades and, ultimately, switch the whole university onto the next semester. The process is overseen in a conference room and any errors due to unforseen data abnormalities are debugged on the spot.

Upkeep of the grades codebase is a challenge, when the code needs to work with technologies it was built on, while also being managed withint the evolving technological landscape that requires continuous integration.
